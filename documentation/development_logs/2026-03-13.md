# Development Log Plan: March 13, 2026

Begin the transition from a rules-based explanation layer to a live LLM-backed explanation provider.

## Planned Work

- choose the first live provider integration path
- implement provider client wiring behind the existing explanation abstraction
- draft structured prompt templates using stored anomaly and correlation evidence
- define fallback behavior when provider calls fail

## Risks To Watch

- explanations may become more fluent but less grounded
- provider integration may blur the evidence boundary if prompts are not disciplined
- API cost and error handling can complicate what is currently a deterministic pipeline stage

## Success Condition

At the end of the session, MacroLens should still be able to generate persisted explanations deterministically, but now with the option to use a real hosted model behind the same interface.
